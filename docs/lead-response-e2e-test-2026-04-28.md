# LeadOps Studio — E2E smoke test

Дата: 2026-04-28

## Что проверено

### 1. Путь посетителя на лендинге

Сценарий: потенциальный клиент из ниши мебель/кухни/шкафы открывает лендинг, оставляет demo-заявку на странице.

Проверка выполнена локально через headless Chrome на `demo/index.html`.

Результат:

- форма отправляется без ошибок;
- блок результата становится видимым;
- клиент видит понятный автоответ и список уточняющих вопросов;
- владелец видит карточку заявки с именем, контактом, запросом и рекомендацией;
- событие `landing_demo_submit` пишется в `window.leadOpsEvents` и `localStorage.leadops_events`;
- `LEADOPS_YM_COUNTER_ID = 0`, поэтому внешние запросы в Метрику пока не отправляются.

Тестовые данные:

- имя: Иван;
- контакт: `+79990000000`;
- запрос: “Нужен шкаф в прихожую, хочу понять цену и сроки”.

Итог: локальный demo-flow работает.

### 2. Путь посетителя из лендинга в Telegram-бота

Сценарий: посетитель нажимает CTA “Получить demo в Telegram” и приходит в бота по deep link `/start landing_demo`.

Проверка выполнена мок-тестом без обращений к Telegram API и без записи в реальные lead-файлы.

Результат:

- `/start landing_demo` распознаётся как источник `landing_demo`;
- бот сразу запускает flow, без лишнего welcome-экрана;
- первая ниша в списке — `ремонт / кухни / окна`, клиники смещены ниже;
- после 6 ответов формируется demo-preview;
- лид скорится как `горячий`, score `7`;
- пишутся события:
  - `bot_deeplink_start`;
  - `bot_flow_start`;
  - `bot_lead_finished`;
  - `bot_qualified_lead`.

Итог: bot-side E2E сценарий работает в локальном тесте.

## Что поправлено по итогам теста

1. Deep link `landing_demo` теперь сразу запускает demo-flow, а не показывает промежуточный welcome. Меньше трения для посетителя.
2. В списке ниш бота первой поставлена `ремонт / кухни / окна`; клиники/косметология оставлены ниже, так как этот ICP временно отложен.

Файл изменений:

- `scripts/leadops_bot.py`

## Наблюдения как “потенциальный клиент”

Сильные стороны:

- оффер понятен: быстрый первый ответ и карточка заявки;
- demo на странице быстро показывает механику;
- для ремонта/кухонь формулировки попадают в боль “клиент ушёл к тому, кто ответил первым”;
- карточка владельцу выглядит полезно и не обещает магии.

Слабые места:

- публичная GitHub Pages-версия может отставать от локальной, если не запушить последние изменения;
- demo-форма на странице говорит “данные никуда не отправляются”, поэтому для реального лида нужен отдельный CTA в Telegram/форму;
- в боте нет сбора контакта внутри flow — для настоящего лида Telegram chat_id есть, но явного телефона/сайта нет;
- нет отдельного короткого блока “что получите через 10 минут” перед CTA.

## Рекомендации до четверга

1. Не запускать рекламу и не писать людям.
2. Подготовить следующий безопасный шаг: добавить в bot-flow вопрос “оставьте сайт/ссылку/контакт для разбора” после qualified lead.
3. После подтверждения — запушить локальные изменения и проверить публичную страницу.
4. Перед платным трафиком заменить `LEADOPS_YM_COUNTER_ID = 0` на ID счётчика Метрики и создать цели.
