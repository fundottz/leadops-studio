# Operating model: LeadOps Studio как виртуальная команда

Дата: 2026-04-27

## Решение Андрея

Андрей не должен быть фронтовым исполнителем на раннем этапе.

Роли:

- Андрей — CEO / владелец бизнес-проекта / принимающий решения.
- Агент — тимлид / проектный менеджер / операционный lead, который ведёт запуск бизнеса.
- Подагенты/инструменты — виртуальная команда: ресерч, outreach, лендинг, MVP, аналитика, follow-up.

Цель: Андрей подключается только там, где реально нужен человек: стратегическое решение, подтверждение внешнего действия, пилот, деньги, сложный созвон.

## Коммуникационная модель

Стартуем с Telegram, email откладываем.

Telegram нужен для:

- входящих обращений;
- первичной квалификации;
- записи на demo/разбор;
- уведомлений агенту/Андрею;
- передачи тёплого лида человеку.

Email оставляем на потом:

- для домена;
- для документов;
- для компаний, где Telegram неуместен;
- для платных пилотов и договорённостей.

## Важное ограничение Telegram Bot API

Telegram-бот не может первым написать человеку/компании в личку, если пользователь сам не начал диалог с ботом.

Значит:

- бот хорош как входящая точка и квалификатор;
- для холодного outbound в Telegram нужен отдельный рабочий user-account с номером;
- либо outbound идёт через сайты/формы/email/публичные контакты, а CTA ведёт в бота.

## Рекомендуемая архитектура на старт

### Минимальная версия

1. Создать Telegram-бота LeadOps Studio через BotFather.
2. Бот принимает входящие обращения по ссылке `t.me/<bot_username>`.
3. Outreach-сообщения ведут в бота или на demo-link.
4. Агент ведёт первичную коммуникацию через бота до момента, где нужен Андрей.
5. Андрей получает только эскалации:
   - человек хочет созвон;
   - есть интерес к пилоту;
   - вопрос про деньги/договорённости;
   - нестандартный запрос.

### Для холодного Telegram outreach

Нужен отдельный рабочий Telegram user-account:

- отдельный номер;
- нейтральное имя LeadOps Studio / LeadOps Team;
- использовать аккуратно, без спама;
- бот остаётся точкой квалификации и обработки входящих.

## Эскалация Андрею

Агент ведёт сам:

- ответы на типовые вопросы;
- квалификацию;
- отправку demo-flow;
- сбор вводных;
- follow-up;
- подготовку к созвону.

Андрея звать, если:

- есть запрос на пилот;
- человек готов обсуждать оплату;
- нужен живой созвон с доверием;
- требуется решение по цене/условиям;
- появился потенциальный партнёр.

## Следующий шаг

Создать Telegram-бота.

Нужно от Андрея:

- username бота;
- bot token из BotFather;
- подтверждение, что можно подключить бота к OpenClaw как отдельный Telegram account/канал для проекта.

После подключения:

- настроить welcome-message;
- сценарий квалификации;
- уведомления/эскалации;
- ссылку на бота в outreach;
- таблицу входящих лидов.
