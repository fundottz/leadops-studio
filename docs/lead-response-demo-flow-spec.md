# Demo-flow specification: автоответ на лид

## Цель MVP

Показать потенциальному клиенту живой сценарий, не полноценный продукт.

Минимальная демонстрация должна доказывать:

- заявка не теряется;
- клиент получает ответ быстро;
- менеджер получает структурированный лид;
- сценарий легко адаптировать под нишу.

## Версия 0: ручной/полуавтоматический MVP

### Компоненты

1. Простая форма заявки
2. Webhook/API обработчик
3. Telegram bot для уведомлений менеджеру
4. Таблица/хранилище заявок
5. Скрипт вопросов по нише

### Поток

1. Пользователь оставляет заявку:
   - имя;
   - телефон/Telegram;
   - услуга;
   - комментарий.

2. Система отправляет автоответ:
   - “Спасибо, получили заявку. Чтобы быстрее передать специалисту, уточните 3 вопроса.”

3. Бот задаёт вопросы.

4. Ответы собираются в одну карточку.

5. Менеджер получает уведомление:
   - новый лид;
   - ниша/услуга;
   - ответы;
   - приоритет;
   - ссылка/контакт клиента;
   - рекомендуемый следующий шаг.

6. Заявка сохраняется в таблицу.

## Сценарий: стоматология/клиника

### Автоответ

Здравствуйте! Получили вашу заявку. Чтобы администратор быстрее подобрал время и специалиста, уточните, пожалуйста, несколько деталей.

### Вопросы

1. Какая услуга интересует?
   - консультация
   - лечение
   - имплантация
   - ортодонтия
   - гигиена
   - другое

2. Насколько срочно нужно записаться?
   - сегодня/завтра
   - на этой неделе
   - в течение месяца
   - просто узнаю

3. Какой район/филиал удобнее?

4. Как удобнее связаться?
   - звонок
   - Telegram/WhatsApp

5. Удобное время для связи?

### Уведомление менеджеру

Новая заявка: стоматология

Услуга: имплантация
Срочность: на этой неделе
Район: север/центр
Связь: Telegram
Время: после 18:00
Контакт: @username / телефон

Рекомендация: горячий лид, ответить до 5 минут.

## Сценарий: ремонт/кухни/окна

### Автоответ

Здравствуйте! Получили заявку. Чтобы менеджер быстрее сориентировал по срокам и стоимости, уточните, пожалуйста, несколько деталей.

### Вопросы

1. Что нужно сделать?
   - ремонт квартиры
   - кухня на заказ
   - окна
   - санузел
   - дизайн-проект
   - другое

2. Где объект?
   - город/район

3. Когда планируете начать?
   - срочно
   - в течение месяца
   - через 1–3 месяца
   - пока сравниваю варианты

4. Есть примерный бюджет?
   - до 300 тыс ₽
   - 300–700 тыс ₽
   - 700 тыс – 1,5 млн ₽
   - 1,5 млн ₽+
   - пока не знаю

5. Есть фото/план/размеры?

6. Как удобнее связаться?

### Уведомление менеджеру

Новая заявка: ремонт/кухня

Тип: кухня на заказ
Локация: Москва, ЮЗАО
Старт: в течение месяца
Бюджет: 700 тыс – 1,5 млн ₽
Материалы: есть план
Связь: WhatsApp
Контакт: телефон

Рекомендация: приоритет высокий, клиент сравнивает подрядчиков.

## Приоритет лида

Простая скоринговая логика:

Горячий:

- срочно / сегодня / на этой неделе;
- есть бюджет;
- оставил телефон;
- выбрал конкретную услугу.

Тёплый:

- срок в течение месяца;
- услуга понятна;
- бюджет неизвестен.

Холодный:

- просто узнаю;
- нет контакта;
- нет конкретной услуги.

## Демо для продажи

Показывать так:

1. “Вот форма, которую видит клиент.”
2. “Вот что клиент получает через 1–2 минуты.”
3. “Вот какие вопросы задаём.”
4. “Вот карточка, которую получает менеджер.”
5. “Вот как это можно подключить к вашей CRM.”

## Что не делать в MVP

- не строить личный кабинет;
- не делать сложный AI-диалог;
- не обещать медицинские/юридические консультации;
- не интегрировать все CRM до первого клиента;
- не автоматизировать edge cases.

## Следующее техническое решение

Для первого реального пилота выбрать один из вариантов:

### Быстро и no-code

Tilda/форма → webhook → n8n/Make → Telegram + Google Sheets

Плюсы: быстро.
Минусы: зависимость от сервисов, меньше контроля.

### Лёгкий backend

HTML/форма → FastAPI/Node → Telegram Bot API → SQLite/Google Sheets

Плюсы: контроль, можно переиспользовать.
Минусы: чуть дольше.

Рекомендация: если нужен демо за 1 день — no-code. Если хотим основу для повторяемого бизнеса — лёгкий backend.
