# Operating rules

## Git commits

Правило: регулярно фиксировать проект в git при достижении логических вех.

Коммитить после:

- завершения смыслового артефакта: лендинг, bot-flow, бюджетная модель, advisory review;
- изменения стратегии/воронки;
- подготовки пакета к внешнему запуску;
- подключения нового канала/бота/аналитики;
- заметного изменения data/shortlist;
- перед рискованными изменениями и после них, если результат успешен.

Не коммитить:

- сырые секреты, tokens, .env;
- личные данные и приватные контакты;
- временные debug-файлы;
- бессмысленные микроправки без законченного состояния.

Формат сообщений:

- `docs: ...`
- `demo: ...`
- `bot: ...`
- `marketing: ...`
- `data: ...`
- `ops: ...`

После коммита по возможности делать push в `origin/main`.
