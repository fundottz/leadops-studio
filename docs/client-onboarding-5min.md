# 5-минутное подключение нового клиента

Цель: быстро показать клиенту вау-эффект — отдельный бот под его бренд, заявки падают в его Telegram-чат, сценарий вопросов настроен под нишу.

## Архитектура

На старте используем модель:

```text
@LeadOpsStudioBot  -> demo / продажи Заявки CRM
@VasyaKuhniBot     -> клиентский бот «Кухни Васи»
@OknaIvanBot       -> клиентский бот «Окна Иван»

все боты -> один backend scripts/leadops_bot.py -> разные tenant configs -> разные manager chats
```

Почему отдельный бот на клиента:

- клиент видит бренд бизнеса, а не LeadOps;
- проще доверие и конверсия;
- меньше риск перепутать заявки;
- проще объяснить владельцу: «это ваш бот для приёма заявок».

## Что нужно от клиента

1. Название бизнеса.
2. Ниша / услуга.
3. 4–6 вопросов, которые он обычно задаёт клиенту.
4. Telegram-чат, куда должны падать заявки.
5. Точки входа: сайт, Авито, Яндекс Карты, профиль Telegram/WhatsApp, QR.

## Подключение за 5 минут

### 1. Создать бота в BotFather

В Telegram открыть `@BotFather`:

```text
/newbot
Кухни Васи — заявки
VasyaKuhniBot
```

Скопировать token. Token не коммитить.

### 2. Создать чат заявок

Создать Telegram-группу:

```text
Кухни Васи — заявки
```

Добавить туда владельца/менеджера и нового бота.

### 3. Получить manager_chat_id

Временно запустить бота с конфигом клиента или старым single-bot режимом, затем в группе написать:

```text
/whoami
```

Бот ответит:

```text
chat_id: -1001234567890
```

Это значение вставить в `manager_chat_id`.

### 4. Добавить клиента в локальный конфиг

Скопировать пример:

```bash
cp data/clients.example.json data/clients.local.json
```

В `data/clients.local.json` добавить блок клиента:

```json
{
  "tenant_id": "vasya_kitchens",
  "flow_type": "client_intake",
  "bot_token": "BOTFATHER_TOKEN",
  "manager_chat_id": "-1001234567890",
  "copy": {
    "company_name": "Кухни Васи",
    "manager_card_title": "Новая заявка — Кухни Васи",
    "welcome_title": "Здравствуйте! Это быстрый приём заявок «Кухни Васи».",
    "welcome_text": "Уточню несколько деталей по кухне или шкафу, чтобы мастер быстрее сориентировал по срокам, замеру и стоимости.",
    "handoff_text": "Спасибо, заявку передали мастеру. Он вернётся с ответом по срокам и следующему шагу.",
    "client_finish_text": "Спасибо, заявку приняли. Передали мастеру — он вернётся с ответом по срокам, замеру и следующему шагу."
  },
  "questions": [
    {"key": "need", "text": "1/5. Что нужно рассчитать?", "options": ["кухня", "шкаф", "гардеробная", "другое"]},
    {"key": "measurements", "text": "2/5. Есть размеры, фото или план?", "options": ["есть размеры", "есть фото/пример", "пока нет", "нужен замер"]},
    {"key": "timeline", "text": "3/5. Когда нужен замер или расчёт?", "options": ["сегодня/завтра", "на этой неделе", "в течение месяца", "пока прицениваюсь"]},
    {"key": "district", "text": "4/5. В каком районе объект?", "options": ["Москва", "Московская область", "другой город", "напишу в сообщении"]},
    {"key": "contact", "text": "5/5. Как удобнее продолжить?", "options": ["написать в Telegram", "созвониться", "прислать расчёт", "записаться на замер"]}
  ]
}
```

### 5. Проверить токен и команды

```bash
./scripts/run-bot.sh --clients-config data/clients.local.json --tenant vasya_kitchens --check
./scripts/run-bot.sh --clients-config data/clients.local.json --tenant vasya_kitchens --set-commands
```

### 6. Запустить всех клиентских ботов

```bash
./scripts/run-bot.sh --clients-config data/clients.local.json
```

Для продакшена этот запуск нужно завернуть в systemd/supervisor. На MVP можно поднять вручную или через текущий процесс-раннер.

## Ссылки для размещения

Для каждой точки входа используем deep link с source:

```text
https://t.me/VasyaKuhniBot?start=site_hero
https://t.me/VasyaKuhniBot?start=site_contacts
https://t.me/VasyaKuhniBot?start=avito_profile
https://t.me/VasyaKuhniBot?start=telegram_post
https://t.me/VasyaKuhniBot?start=manual_reply
```

Кнопки должны называться не «Открыть бота», а действием клиента:

- Рассчитать кухню
- Записаться на замер
- Получить быстрый ответ
- Отправить размеры для расчёта
- Узнать сроки изготовления

## Что показывать клиенту на продаже

Демо-скрипт:

1. «Создаём вашего бота под бренд».
2. «Вот ссылка `t.me/VasyaKuhniBot?start=site_hero`».
3. «Клиент отвечает на 5 вопросов».
4. «Вам в чат падает готовая карточка заявки».
5. «Можно ответить клиенту reply на карточку или командой `/reply chat_id текст`».

## Что уже умеет backend

- несколько ботов в одном процессе через `--clients-config`;
- отдельное состояние/лиды/аналитика на tenant: `data/tenants/<tenant_id>/`;
- отдельные вопросы на клиента;
- отдельные тексты и бренд клиента;
- режим `client_intake`, где клиент видит обычный финал заявки, а не demo-продажу LeadOps;
- отдельный `manager_chat_id`;
- `/whoami` для быстрого получения chat_id;
- reply из manager-чата клиенту от имени клиентского бота;
- deep links с source tracking.

## Ограничения MVP

- BotFather token создаётся вручную.
- Нет web-admin панели: конфиг пока JSON.
- Telegram-боты отдельные, но backend общий.
- WhatsApp/Avito/CRM — следующий этап после первого рабочего клиента.
