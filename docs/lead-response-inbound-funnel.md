# Inbound funnel: лендинг → Telegram-бот → квалификация → встреча

Дата: 2026-04-27
Статус: выбранная рабочая схема запуска

## Решение

Рабочая схема проверки теперь двухшаговая:

1. **Сначала тёплая discovery-проверка**: 3–5 контактов из ремонта / кухонь / окон / стройки / смежных интро, чтобы подтвердить боль speed-to-lead без расходов и без холодной рассылки.
2. **Затем inbound-воронка**: лендинг → Telegram-бот → квалификация → эскалация тёплого лида Андрею.

Inbound-воронка нужна для масштабирования после первичного сигнала:

1. Трафик идёт на лендинг.
2. Лендинг тестирует оффер и боль.
3. CTA ведёт в Telegram-бота.
4. Бот прогревает, квалифицирует и регистрирует лида.
5. Если лид тёплый — бот эскалирует Андрею и предлагает встречу/звонок/переписку.
6. Андрей подключается только к тёплым возможностям.

Причина: платный трафик имеет смысл запускать только после короткой ручной проверки боли и подключённой аналитики, чтобы не сжечь бюджет на непроверенный угол.

## Архитектура воронки

### 1. Трафик

Стартовые каналы после тёплой проверки:

- Яндекс.Директ Поиск — горячие запросы по ремонту / кухням / окнам / стройке и B2B speed-to-lead;
- РСЯ — боли собственников/РОПов: “заявки теряются”, “менеджеры отвечают поздно”;
- ретаргетинг — догрев посетителей лендинга;
- позже: Telegram Ads / посевы / партнёрские каналы.

Клиники / стоматология / косметология пока не первый paid-тест без тёплого интро или отдельного подтверждения.

### 2. Лендинг

Цель лендинга:

- объяснить боль за 5–10 секунд;
- не продавать сложную автоматизацию;
- довести до действия: “получить схему / посмотреть demo / разобрать воронку”.

Главный CTA:

> Получить схему автоответа в Telegram

Альтернативные CTA:

- Посмотреть demo-flow
- Разобрать, где теряются заявки
- Посчитать потери заявок

### 3. Telegram-бот

Роль бота:

- принять входящий лид;
- объяснить оффер простыми словами;
- уточнить нишу и объём заявок;
- понять боль;
- показать demo/пример сценария;
- предложить следующий шаг;
- сохранить лид;
- эскалировать тёплую возможность.

### 4. Эскалация

Бот эскалирует Андрею, если:

- есть 20+ входящих заявок в месяц;
- есть сайт/реклама/CRM/мессенджеры;
- человек признаёт проблему скорости ответа/потерь;
- хочет demo/разбор;
- спрашивает цену/пилот;
- готов к созвону.

Формат эскалации:

```
Тёплый лид LeadOps Studio

Компания/ниша: ...
Заявок в месяц: ...
Каналы: ...
Проблема: ...
CRM: ...
Интерес: demo / пилот / цена / созвон
Контакт: ...
Рекомендация: подключиться / назначить звонок / продолжить ботом
```

## Bot flow

### /start

Здравствуйте! Это LeadOps Studio.

Мы помогаем бизнесу не терять входящие заявки: клиент получает ответ за 1–2 минуты, система собирает 3–5 вводных и передаёт менеджеру готовый лид в Telegram или CRM.

Что хотите сделать?

1. Посмотреть demo
2. Разобрать мою воронку
3. Понять стоимость пилота

### Квалификация

1. Какая у вас ниша?
2. Сколько входящих заявок примерно в месяц?
3. Откуда приходят заявки?
4. Как быстро сейчас отвечаете?
5. Есть CRM? Какая?
6. Что болит сильнее?
   - долго отвечают менеджеры;
   - заявки ночью/в выходные;
   - много мусорных лидов;
   - нет контроля обработки;
   - нужно связать сайт/мессенджеры/CRM.
7. Хотите посмотреть пример сценария под вашу нишу?

### Дожим

Если лид релевантный:

> Похоже, у вас есть смысл проверить 2-недельный пилот: автоответ + 3–5 вопросов + уведомление менеджеру. Могу предложить короткий разбор на 15 минут и показать сценарий под вашу нишу.

CTA:

- Да, хочу разбор
- Сначала пришлите demo
- Сколько стоит пилот?

## Бюджет теста

Подробная модель бюджета: `notes/lead-response-budget-model.md`.

Рекомендованный регулярный минимальный бюджет:

### Бережный старт

30–50 тыс ₽ / месяц

Что тестируем:

- 1 лендинг;
- 2–3 оффера;
- Поиск по горячим запросам;
- небольшой РСЯ/ретаргетинг.

Цель:

- понять CTR/CR;
- получить первые входящие обращения;
- проверить, умеет ли бот квалифицировать.

### Нормальный тест

80–120 тыс ₽ / месяц

Что тестируем:

- 2 посадки/первых экрана;
- отдельные кампании под ремонт / кухни / окна / стройку и общий B2B speed-to-lead;
- Поиск + РСЯ + ретаргетинг;
- больше статистики по CPL.

Цель:

- 5–15 лидов/месяц;
- 2–5 квалифицированных разговоров;
- 1 пилот.

### Рекомендация

Начать с **30–50 тыс ₽ на первый месяц**, если лендинг и бот готовы.

Не лить 100k+ до проверки связки:

> ключ/объявление → лендинг → бот → квалифицированный лид.

## Метрики

### Лендинг

- CTR объявлений;
- CR лендинг → старт бота;
- стоимость старта бота;
- глубина скролла/клики CTA, если подключена аналитика.

### Бот

- start → ответ на первый вопрос;
- completion rate квалификации;
- доля тёплых лидов;
- стоимость квалифицированного лида;
- доля эскалаций Андрею;
- конверсия в demo/созвон/пилот.

### Бизнес

- CPL;
- CPQL — стоимость квалифицированного лида;
- cost per demo;
- cost per pilot;
- окупаемость пилота.

## Следующие действия

1. Создать Telegram-бота LeadOps Studio.
2. Подключить бота к OpenClaw или сделать промежуточный bot handler.
3. Обновить лендинг: главный CTA → Telegram bot.
4. Подготовить UTM и события аналитики.
5. Запустить минимальный Директ на 30–50 тыс ₽ после подтверждения.
6. Первую неделю смотреть не только CPL, а качество диалогов в боте.

## Решения, которые нужны от Андрея

1. Создать bot token через BotFather.
2. Подтвердить минимальный бюджет: 30–50 тыс ₽ / месяц.
3. Подтвердить, что первый канал — Яндекс.Директ.
4. Подтвердить формат эскалации: сообщение Андрею в личку, когда лид тёплый.
